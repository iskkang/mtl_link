## 1. 프로젝트 개요

### 프로젝트명
**MTL Link** (이하 *Messenger*)

### 목적
회사 내부 직원들이 별도 웹사이트에 로그인하여 1:1 채팅과 그룹 채팅을 하고, 텍스트·이미지·문서·링크를 주고받을 수 있는 **웹 기반 사내 메신저**를 구축한다.

### 핵심 방향
- WhatsApp, Telegram, 카카오톡과 연동하지 않는다.
- MoveTalk 시스템과는 **별도 인프라**로 운영한다 (DB·Auth·Storage 모두 분리).
- MoveTalk 헤더에 **"메신저 열기"** 버튼을 추가해 새 탭에서 연다. 임베드하지 않는다.
- 메신저는 *항상 켜둔 채 작업하는 도구*이므로 독립 사이트로 운영한다.

---

## 2. 개발 배경

회사 업무에서 다음 커뮤니케이션 수요가 존재한다.

- 직원 간 빠른 업무 대화
- 화물·통관·견적·수금·선적 관련 자료 공유
- 이미지·캡처·PDF·Word·Excel·PowerPoint 파일 공유
- 업무별/팀별 대화방 관리
- 대화 기록과 첨부파일의 체계적 보관
- **다국어 직원 간 언어 장벽 해소** (한국어·러시아어·영어 혼용 환경)
- 향후 AI 번역, 요약, 업무 이슈 추출 기능 확장

기존 외부 메신저는 편리하지만 회사 업무 데이터가 외부 플랫폼에 분산되고, 업무 흐름에 맞춘 확장이 어렵다. 따라서 회사 내부 업무에 맞는 자체 웹 메신저를 구축한다.

---

## 3. 제품 정의

> 직원들이 로그인해서 1:1 또는 그룹 채팅을 하고, 텍스트·이미지·문서·링크를 주고받으며, **음성으로 말하면 상대방 언어로 자동 번역되어 전송**되고, 모든 대화와 첨부파일이 방별로 저장되는 **웹 기반 사내 메신저**.

## 5. 주요 사용 시나리오

### 시나리오 1: 직원 간 1:1 대화
1. 직원 A가 메신저에 로그인한다.
2. 직원 목록에서 직원 B를 선택한다.
3. 1:1 채팅방이 생성되거나 기존 방이 열린다.
4. 메시지를 입력하고 전송한다.
5. 직원 B의 화면에 실시간으로 메시지가 표시된다.

### 시나리오 2: 그룹방 업무 대화
1. 관리자 또는 일반 사용자가 새 그룹방을 만든다.
2. 참여 직원을 초대한다.
3. 그룹방에서 업무 메시지와 첨부파일을 공유한다.
4. 방 목록은 최근 메시지 기준으로 정렬된다.

### 시나리오 3: 파일 첨부
1. 직원이 메뉴바의 파일 버튼을 누르거나 채팅 영역에 드래그앤드롭한다.
2. PDF, DOCX, XLSX, PPTX, 이미지 등을 선택한다.
3. 파일이 업로드되고 채팅 메시지로 표시된다.
4. 다른 직원은 파일을 다운로드하거나 이미지 썸네일을 클릭해 확인한다.

### 시나리오 4: 링크 공유
1. 직원이 선사 트래킹 링크 또는 외부 URL을 메시지에 붙여넣는다.
2. URL이 자동으로 클릭 가능한 링크로 표시된다. 클릭 시 새 탭에서 열린다.

### 시나리오 5: 음성 번역 메시지 (v2.1)
1. 직원 A(한국어 사용자)가 홍길동(영어 설정) 채팅방에서 마이크 버튼을 누른다.
2. "내일 부산항에 화물 도착합니다"라고 말한다.
3. 정지 버튼을 누르면 1~2초 내에 영어로 번역되어 메시지가 전송된다.
4. 홍길동 화면에는 "The shipment will arrive at Busan port tomorrow" 로 표시된다.
5. 홍길동이 "원본 보기"를 클릭하면 한국어 원문도 확인할 수 있다.

### 시나리오 6: 신규 직원 합류 (관리자 동선)
1. 관리자가 `/admin` 페이지에서 *"직원 추가"*를 클릭한다.
2. 이름, 이메일, 부서, 직급을 입력한다.
3. 시스템이 임시 비밀번호를 생성하고 가입 완료 메일을 자동 발송한다.
4. 신규 직원은 첫 로그인 시 비밀번호 변경을 요구받는다.

> 자가 가입(self sign-up)은 차단한다. Supabase Auth의 `enable_signup = false`.

## 7. 음성 번역 메시지 — v2.1

### 7.1 전체 흐름
```text
마이크 버튼 클릭 → 녹음 시작 (최대 60초)
→ 정지 버튼 클릭
→ Edge Function: Whisper STT → Claude Haiku 번역
→ 음성 파일 메모리에서 즉시 폐기 (저장 안 함)
→ 번역된 텍스트를 메시지로 전송
→ 메시지 버블: 번역문 + "원본 보기" 토글
```

### 7.2 서비스 구성

| 역할 | 서비스 | 이유 |
|---|---|---|
| STT | OpenAI Whisper API | 한국어·러시아어·영어 정확도 우수 |
| 번역 | Anthropic Claude Haiku | 비즈니스 어조 자연스러움, 저렴 |

### 7.3 녹음 제약
- 최대 60초. 초과 시 자동 정지 + 토스트 안내
- 코덱: webm/opus (브라우저 기본)
- **음성 파일은 Edge Function 메모리에서만 처리 → DB·Storage 어디에도 저장 안 함**

### 7.4 메시지 표시
- 번역문 우선 표시
- "원본 보기" 토글 버튼 → 클릭 시 원본 + 번역 함께 표시
- 메시지 하단에 `🎤 음성 번역 · KO → EN` 표시

## 8. 수신자별 번역 언어 설정 — v2.1

### 8.1 설정 방식
- 방 헤더의 *"번역 언어: English ↗"* 클릭 → 모달

### 8.2 지원 언어
`한국어 (ko)` / `English (en)` / `Русский (ru)` / `中文 (zh)` / `日本語 (ja)` / `O'zbek (uz)` / `번역 안 함`

### 8.3 결정 우선순위
1. 발신자가 수신자별로 설정한 언어 (`translation_preferences` 테이블)
2. 수신자의 기본 언어 (`profiles.preferred_language`)
3. 기본값: 영어

### 8.4 그룹방
그룹방은 owner가 방 단위 default 언어를 설정. 개인 override는 v2.2에서 검토.

## 15. 보안 원칙

- 비로그인 사용자는 모든 페이지 차단
- 사용자는 자신이 참여한 방의 데이터만 조회·전송 (RLS)
- 파일은 private Storage + Signed URL (60분 만료)
- 실행파일·스크립트 업로드 차단 (확장자 + MIME 이중 검증)
- Service Role Key는 프론트엔드에 절대 노출 금지 (Edge Function 전용)
- OpenAI·Anthropic API Key도 Edge Function 시크릿에만 보관
- 자가 가입 차단 (관리자 초대 방식)
- 첫 로그인 시 비밀번호 변경 강제
- **음성 파일은 Edge Function 메모리에서만 처리. 외부 저장 없음**
- 민감 정보 음성 번역 사용 금지 (운영 가이드 명시)
