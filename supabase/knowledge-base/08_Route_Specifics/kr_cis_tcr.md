# Korea → CIS (TCR 노선 가이드)

## 1. 노선 기본 정보

### 서비스 유형

| 서비스 | 구간 | 운송 방식 |
|---|---|---|
| TCR ALL RAIL | Incheon/Busan → Qingdao/Lianyungang → Khorgos/Alashankou → Altynkol/Dostyk → CIS | 해상 + 철송 |
| TCR RAIL+TRUCK | Incheon/Busan → Qingdao → Kashgar → CIS | 해상 + 철송 + 트럭 |
| TCR ALL TRUCK | Incheon/Busan → Qingdao → Kashgar → CIS | 해상 + 트럭 |

### 주요 경유 국경

```
중국 국경: Khorgos (霍尔果斯) / Alashankou (阿拉山口)
카자흐스탄 국경: Altynkol / Dostyk
```

## 2. 부킹 전 필수 확인 항목

- [ ] 컨테이너 사이즈/수량
- [ ] 화물 수량/중량/CBM
- [ ] **중량화물 여부** — 패킹당 중량 확인
- [ ] **위험물(DG) 여부** — 위험물/비위험물 구분
- [ ] **영문 아이템명 + HS CODE** — 진행 가능 여부 사전 확인
- [ ] 포장 단위
- [ ] **비철금속 여부** — 비철금속은 운임 높음, 별도 확인
- [ ] 운임 지불처, 통관 여부, 내륙 운송 필요 여부

## 3. 진행 프로세스 (NOMI 건)

```
1. 현지 지사/법인으로부터 부킹 요청 수신
   ↓
2. 수출자에게 출하일정, 화물 디테일, 운임 조건 확인
   ↓
3. 선사 스케줄 + TCR 진행 가이드 안내
   ↓
4. 선사 부킹 진행 (★SOC 부킹 필수★)
   부킹 정보 수출자에게 안내
   ↓
5. 수출자로부터 서류(CI/PL/면장) + 작업사진 수령
   → 선사 S/R 마감
   ↓
6. 수출자에게 HBL 전달 및 확인 요청
   → 중국 AGENT에 MBL/HBL/CI/PL + Empty Return Instruction + 작업사진 전달
   ↓
7. 선적일: 선사 MBL SURRENDER 또는 SEA WAYBILL 처리
   → 중국 측 전달
```

> **CI/PL**: 수입자용 최종 서류
> **Empty Return Instruction**: SOC 컨테이너만 해당 (매각처 문의, 우즈벡 건은 우즈벡에서 제공)

## 4. 필수 제출 서류

| 서류 | 필수 여부 | 비고 |
|---|---|---|
| MBL | ✅ 필수 | 컨테이너당 발행 |
| HBL | ✅ 필수 | |
| Commercial Invoice | ✅ 필수 | 영문 + 러시아어 + HS CODE |
| Packing List | ✅ 필수 | 영문 + 러시아어 + HS CODE |
| 작업사진 | ✅ 필수 | 최소 8장 (TCR 기준) |
| Railway Bill | ✅ 필수 | 컨테이너당 1부 |
| Certificate / MSDS | 조건부 | 해당 품목만 |

## 5. 중량 제한

| 서비스 | VGM 기준 | 최대 VGM |
|---|---|---|
| TCR ALL RAIL | 30톤 미만 | 30톤 (가능하면 29톤 미만) |
| TCR RAIL+TRUCK | 23톤 기준 | 30톤 |
| TCR ALL TRUCK | 23톤 기준 | 별도 확인 |

- 패킹당 중량 1,000kg 이상 → 철도역 체크 필요
- 패킹당 중량 5톤 미만 선적 가능 (사전 컨펌 필수)
- 중량화물은 포장 타입 사전 확인 필수

### 편중 제한

| 항구 | 전후 편차 허용 |
|---|---|
| 청도(Qingdao) | 5톤 이하 |
| 연운항(Lianyungang) | 3톤 이하 |

## 7. 자주 발생하는 이슈

| 이슈 | 빈도 | 예방 |
|---|---|---|
| CI/PL 품명 포괄적 ("Machine Parts") → 중국 세관 검사 | 높음 | 출발 전 품명 구체화 확인 |
| 작업사진 누락 → 중국 반송 위험 | 중간 | 사전 체크리스트 화주 전달 |
| 편중 발생 → INSPECTION + 재작업 | 중간 | 편중 기준 사전 교육 |
| HS코드 미확인 → 통관 지연 | 높음 | 부킹 전 HS코드 확인 필수 |
| SOC 부킹 미진행 → Empty Return 문제 | 낮음 | 부킹 시 SOC 여부 확인 |
