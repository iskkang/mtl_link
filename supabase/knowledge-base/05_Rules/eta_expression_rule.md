# ETA Expression Rule

## 목적

ETA(Estimated Time of Arrival) 표현 시 **약속과 추정을 분리**하기 위한 표준 규칙.
화주에게 잘못된 약속을 하면 클레임으로 직결되므로, 운영 AI와 직원 모두 본 규칙을 따른다.

## 핵심 원칙

**ETA는 절대 단정하지 않는다.**

ETA는 본질적으로 추정치이며, 운송·통관·기상·항만 운영 등 여러 변수에 영향받는다.
따라서 **모든 ETA 언급에는 단서가 붙어야 한다.**

## ETA 표현 등급

| 상황 | 표현 |
|---|---|
| **확정 도착** (이미 도착, 도착 직전) | "X월 X일 도착 완료" / "X월 X일 도착 예정 (도착지 ETA 확정)" |
| **운송 중 정상** | "현재 ETA는 X월 X일이며, 운송 상황에 따라 변경 가능합니다" |
| **운송 중 변동 가능** | "현재 정보 기준 ETA는 X월 X일 예상이며, [요인]에 따라 변경 가능성이 있습니다" |
| **운송 중 지연 발생** | "지연 발생으로 기존 ETA 변경이 필요하며, 현재 새로운 ETA는 X월 X일 예상입니다 (변경 가능)" |
| **사유 미확인 상황** | "정확한 ETA는 현재 확인 중이며, 확인 즉시 안내드리겠습니다" |

## 금지 표현

| ❌ 금지 | ✅ 대체 |
|---|---|
| "X일에 도착합니다" | "현재 ETA는 X일이며, 변경 가능성이 있습니다" |
| "내일 도착할 것입니다" | "현재 정보 기준 내일 도착 예상이며, 운송 상황 모니터링 중입니다" |
| "곧 도착합니다" | (구체적 시점 또는 "현재 도착지 X km 지점 운송 중") |
| "분명히 X일 안에" | "현재 ETA 기준 X일 내 도착 예상" |
| "확실히 X일까지" | "이상 없이 진행되면 X일 도착 예상" |
| "걱정하지 마세요. X일에 옵니다" | (단정 표현 자체 금지) |

## 영문 표현

| ❌ 금지 | ✅ 대체 |
|---|---|
| "Will arrive on X" | "Current ETA is X, subject to change" |
| "Arriving tomorrow" | "ETA tomorrow based on current status" |
| "Definitely by X" | "Targeting arrival by X, monitoring closely" |
| "Don't worry, it will arrive" | (단정 표현 자체 금지) |

## ETA 변경 시 대응

ETA가 변경되면 **즉시 화주에게 통보**한다. 변경된 ETA를 숨기고 시간이 흐른 뒤 알리면 클레임 위험이 커진다.

```
변경 ETA 확인 → 1시간 내 화주 통보 → 사유 + 새 ETA + 영향 명시
```

## 화주가 "확정 ETA 달라"고 할 때

화주가 "정확한 ETA"를 요구해도 **단정 표현을 사용하면 안 된다**.

대신 다음을 제공한다:

1. **현재 ETA 범위**: "X월 X일 ~ X월 X일 사이 예상"
2. **변동 요인**: "다음 요인에 영향받을 수 있습니다: 통관 소요 시간, 환적 항만 운영 상황 등"
3. **모니터링 약속**: "변동 시 즉시 안내드리겠습니다"
4. **다음 update 시점**: "다음 위치 도착 시 (예: 환적항 입항 시) 다시 안내드리겠습니다"

화주가 "그래도 확정해달라"고 강요하면 → **운영팀 매니저에게 에스컬레이션**.
임의 약속은 절대 하지 않는다.

## 예외: 확정 가능한 경우

다음의 경우 단정 표현 가능:
- 이미 도착 완료
- 도착지 항만/공항 입항 완료 (창고 입고만 남은 상태)
- 운송사로부터 서면 확정 ETA 수령 (단, 출처 명시)

이 경우에도 안전을 위해 단서 사용 권장:
> "도착지 항만 입항 완료. 창고 입고 예정일 X월 X일 (운송사 확정)"

## 운영 AI 적용

MINT는 ETA 관련 답변 시:

1. **사용자가 ETA를 단정적으로 표현해도** MINT 응답에서는 등급에 맞게 변환한다
2. **화주 메시지 초안 (Section 9)에서는 절대 단정 금지**
3. **추정 ETA는 "Possible Cause" 또는 "Likely" bucket으로 분류**
