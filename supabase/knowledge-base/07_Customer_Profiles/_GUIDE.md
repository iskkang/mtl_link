# 07_Customer_Profiles 가이드

## 목적

고객별 업무 특이사항을 정리하여 MINT가 해당 고객 관련 케이스를 더 정확하게 처리하도록 합니다.

## 포함 내용

- 업무 프로세스 (단계별)
- 고객사 특이사항
- MINT 판단 기준

## 포함하지 않는 내용

- 담당자 연락처 (변경 잦음 → 별도 관리)
- 운임/비용 구조 (민감정보)
- 내부 리펀드/마진 정보 (절대 제외)

아래 고객사는 파악된 정보가 부분적이거나 추가 확인이 필요합니다:

| 고객사 | 현재 파악 내용 | 추가 필요 내용 |
|---|---|---|
| **실리콘투** | 폴란드 DOOR 건, VAT 유예 통관 특이사항 | 전체 프로세스, 통관 방식 확정 후 |
| **그린오션 (도요타통상)** | 광양-그단스크, 트윌 진행 | 전체 프로세스, 운임 구조 제외 버전 |
| **JBG (둔차오토모티브)** | 광양-그단스크, DOOR 건, 하나로 TNS 경유 | 전체 프로세스 |
| **MJA (홍현기 대표)** | TSR FCL, 부산/카오슝/상하이 발 러시아향 | 최신 프로세스 (연락처 제외) |
| **태경산업** | 리가 도착 후 반출 관련 이슈 있었음 | 정상 프로세스 정리 |

## 고객 프로필 작성 양식

새 고객 추가 시 아래 구조 사용:

```markdown
---
doc_type: customer_profile
customer_name: [고객사명]
route: [출발 → 도착]
mode: [sea / rail / sea-rail / truck]
service: [FCL / LCL / TCR / TSR 등]
incoterms: [FOB / CIF / DAP 등]
last_updated: YYYY-MM-DD
---

# [고객사명] 업무 매뉴얼

## 진행 프로세스

```

## 업데이트 주기

- 프로세스 변경 시 즉시 업데이트
- 연 1회 전체 검토
- 담당자 변경 → 연락처는 별도 관리 (본 파일 업데이트 불필요)
